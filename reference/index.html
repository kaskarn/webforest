Function Reference

Complete reference for all webforest functions, organized by category.

Core Functions

The main entry points for creating visualizations. forest_plot() is the primary function for most use cases.

Function Description
forest_plot() Create interactive forest plots
webtable() Create table-only widgets (no forest plot)
split_forest() Create split forest with sidebar navigation
save_plot() Export to SVG, PNG, or PDF
save_split_forest() Export split forest to directory

Styling Functions

Style your plots using the fluent API. These functions work on both forest_plot() widgets and WebSpec objects.

Function Description
set_row_style() Apply row-level styling (bold, italic, color, type, indent)
set_marker_style() Style interval markers (color, shape, opacity, size)
set_column_style() Apply per-cell styling to specific columns
set_theme() Apply a theme by name or object
TipConstructor Arguments vs Fluent API

Every styling function has equivalent direct arguments in forest_plot():

# These produce identical results:
forest_plot(data, ..., row_bold = "is_summary", marker_color = "dir_color")

forest_plot(data, ...) |>
  set_row_style(bold = "is_summary") |>
  set_marker_style(color = "dir_color")

Use direct arguments when configuration is known upfront. Use the fluent API when styling depends on computation or you need multiple variants.
